# BrainLM

BrainLM is a **decoder-only transformer** for brain dynamics (fMRI). It uses block-causal attention over timepoint-major tokens and reconstructs masked signal patches. No proprietary or cohort-specific data is required: you can run pretraining and finetuning on synthetic Arrow datasets generated by the included script.

## Architecture

- **Input**: fMRI recordings with shape `(timepoints, num_parcels)` (e.g. 200 timepoints × 424 AAL parcels). Each parcel has a time series split into patches; parcels are embedded with a learned signal projection plus **xyz spatial** and **sinusoidal temporal** positional encodings.
- **Token order**: Timepoint-major — `[all_parcels_t0, all_parcels_t1, ..., all_parcels_tN]`. A learned CLS token is prepended.
- **Attention**: Block-causal. Parcels at time \(t\) can attend to each other and to all earlier timepoints; they cannot look forward. Implemented with an additive mask passed to `scaled_dot_product_attention`.
- **Masking**: During training, either a fraction of tokens (random masking) or only the last temporal token per parcel (forward masking) is replaced with a learned `[MASK]` embedding. The model predicts the original signal at masked positions.
- **Output**: Reconstruction loss (MSE or MAE) on masked patches. The CLS token can be used for downstream tasks (e.g. regression via an MLP head).

For more detail, see the [Wiki](https://github.com/josueortc/BrainLM/wiki). Wiki source files live in `docs/wiki/` so you can copy them into the GitHub Wiki or read them locally.

## Installation

```bash
git clone https://github.com/josueortc/BrainLM.git
cd BrainLM
pip install -r requirements.txt
# or: pip install -e .
```

Requirements: Python ≥3.8, PyTorch, `transformers`, `datasets`, `numpy`, `scikit-learn`, `matplotlib`, `pyarrow`. Optional: `wandb` for logging.

## Quick start (synthetic data)

Generate fake Arrow datasets and run a short pretraining:

```bash
python generate_sample_data.py --output_dir ./sample_data --num_train 100 --num_val 20
python train.py \
  --output_dir ./runs/demo \
  --train_dataset_path ./sample_data/train \
  --val_dataset_path ./sample_data/val \
  --coords_dataset_path ./sample_data/coords \
  --num_timepoints_per_voxel 200 \
  --timepoint_patching_size 20 \
  --hidden_size 256 \
  --num_hidden_layers 4 \
  --per_device_train_batch_size 2 \
  --max_train_samples 20 \
  --max_eval_samples 10
```

Finetune on a scalar target (e.g. age) using the same data format:

```bash
python finetune.py \
  --model_name_or_path ./runs/demo \
  --train_dataset_path ./sample_data/train \
  --val_dataset_path ./sample_data/val \
  --coords_dataset_path ./sample_data/coords \
  --variable_of_interest_col_name Age.At.MHQ \
  --output_dir ./runs/finetune_demo
```

## Loading from Hugging Face Hub

If a pretrained model is published on the Hub (e.g. `josueortc/brainlm`), load it with:

```python
from transformers import AutoConfig, AutoModelForPreTraining

config = AutoConfig.from_pretrained("josueortc/brainlm", trust_remote_code=True)
model = AutoModelForPreTraining.from_pretrained("josueortc/brainlm", trust_remote_code=True)
```

Or using the package directly:

```python
from brainlm_mae import BrainLMConfig, BrainLMForPretraining

config = BrainLMConfig.from_pretrained("josueortc/brainlm")
model = BrainLMForPretraining.from_pretrained("josueortc/brainlm", config=config)
```

`trust_remote_code=True` is required so that the Hub can resolve the custom `BrainLMConfig` and `BrainLMForPretraining` classes via the config’s `auto_map`.

## Data format

- **Train/validation datasets**: Hugging Face Arrow datasets saved with `dataset.save_to_disk(path)`. Each example must include:
  - A **recording** column (default: `Voxelwise_RobustScaler_Normalized_Recording`): 2D array of shape `(num_timepoints, num_parcels)` (e.g. 500 × 424). The training script randomly crops a contiguous window of `num_timepoints_per_voxel` timepoints (e.g. 200).
  - Optional **metadata** columns for finetuning (e.g. `Age.At.MHQ`, `PHQ9.Severity`).
- **Coordinates dataset**: Arrow dataset with 424 rows (one per parcel) and columns `X`, `Y`, `Z` (e.g. MNI coordinates). Loaded once and broadcast to every sample.

See the [Data Format](https://github.com/josueortc/BrainLM/wiki/Data-Format) Wiki page for the full schema and how to prepare your own data.
